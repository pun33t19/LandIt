Resume Parser
Extract structured data from PDF/DOCX, score ATS compatibility, identify gaps.
Python — parsers/resume_parser.py
import pdfplumber
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from pydantic import BaseModel
from typing import List, Optional

class ResumeData(BaseModel):
    name: str
    email: str
    phone: Optional[str]
    summary: Optional[str]
    skills: List[str]
    experience: List[dict]   # {role, company, duration, bullets}
    education: List[dict]
    certifications: List[str]
    ats_score: int           # 0-100 calculated after parse
    gaps: List[str]          # ["No quantified achievements", ...]

def extract_text_from_pdf(path: str) -> str:
    with pdfplumber.open(path) as pdf:
        return "\n".join(page.extract_text() or "" for page in pdf.pages)

PARSE_PROMPT = ChatPromptTemplate.from_messages([
    ("system", """You are a resume parser. Extract structured data from the resume text.
Return valid JSON matching the ResumeData schema.
For ats_score: score 0-100 based on keyword density, quantified achievements,
action verbs, formatting clarity. For gaps: list specific improvements needed."""),
    ("human", "{resume_text}")
])

async def parse_resume(file_path: str) -> ResumeData:
    llm = ChatOpenAI(model="gpt-4o", temperature=0)
    structured_llm = llm.with_structured_output(ResumeData)
    chain = PARSE_PROMPT | structured_llm
    text = extract_text_from_pdf(file_path)
    return await chain.ainvoke({"resume_text": text})

Resume Optimizer Agent
LangGraph agent that iteratively improves resume quality — rewrites bullets, adds metrics, strengthens summary.
Python — agents/optimizer.py (LangGraph)
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated
import operator

class OptimizerState(TypedDict):
    resume: ResumeData
    suggestions: Annotated[List[str], operator.add]
    improved_bullets: dict
    iteration: int
    approved: bool

def analyze_node(state: OptimizerState):
    """Find weak bullets, passive voice, missing metrics"""
    prompt = f"""Analyze these experience bullets and identify:
1. Bullets missing quantified results (add % / $  / scale)
2. Passive voice ("was responsible for" → "Led")
3. Generic verbs ("worked on" → "Architected", "Reduced")
4. Missing tech keywords for the role
Bullets: {state['resume'].experience}"""
    suggestions = llm.invoke(prompt)
    return {"suggestions": [suggestions.content]}

def rewrite_node(state: OptimizerState):
    """Rewrite bullets applying all suggestions"""
    prompt = f"""Rewrite these resume bullets applying: {state['suggestions']}
Rules: Start with strong action verb. Include metric. ≤2 lines. STAR format.
Original bullets: {state['resume'].experience}
Return JSON: {{"role_company": ["bullet1", "bullet2"]}}"""
    result = structured_llm.invoke(prompt)
    return {"improved_bullets": result, "iteration": state["iteration"] + 1}

def score_node(state: OptimizerState):
    """Re-score ATS after improvements"""
    new_score = calculate_ats_score(state["improved_bullets"])
    return {"approved": new_score >= 80 or state["iteration"] >= 3}

graph = StateGraph(OptimizerState)
graph.add_node("analyze", analyze_node)
graph.add_node("rewrite", rewrite_node)
graph.add_node("score", score_node)
graph.set_entry_point("analyze")
graph.add_edge("analyze", "rewrite")
graph.add_edge("rewrite", "score")
graph.add_conditional_edges("score", lambda s: END if s["approved"] else "analyze")
optimizer = graph.compile()

Job Search Agent
Query job APIs with smart filters, score job-resume match using embeddings, rank by fit.
Filter Parameters
  • Role / title keywords
  • Location + remote/hybrid/onsite
  • Salary range (min/max)
  • Level of experience (LOE)
  • Tech stack must-haves
  • Company size / type
  • Posted within N days
Match Scoring
Embed job description and resume using text-embedding-3-small. Store in Pinecone. Cosine similarity gives a match score (0–1). Combine with keyword overlap and salary fit for a final weighted score shown to user.
Python — tools/job_search_tool.py
import httpx
from langchain.tools import tool
from openai import OpenAI

client = OpenAI()

@tool
async def search_jobs(
    query: str,
    location: str = "remote",
    salary_min: int = None,
    experience_level: str = "mid",  # entry/mid/senior
    employment_type: str = "fulltime",
    tech_stack: List[str] = None,
    num_pages: int = 3
) -> List[dict]:
    """Search jobs using JSearch API (LinkedIn + Indeed + more)"""
    params = {
        "query": f"{query} {' '.join(tech_stack or [])}",
        "location": location,
        "employment_types": employment_type.upper(),
        "job_requirements": experience_level,
        "num_pages": num_pages,
    }
    async with httpx.AsyncClient() as client:
        r = await client.get(
            "https://jsearch.p.rapidapi.com/search",
            params=params,
            headers={"X-RapidAPI-Key": JSEARCH_KEY}
        )
    jobs = r.json()["data"]
    # Filter by salary if provided
    if salary_min:
        jobs = [j for j in jobs
                if j.get("job_min_salary", 0) >= salary_min]
    return [score_job_match(j, resume_embedding) for j in jobs]

def score_job_match(job: dict, resume_embedding: List[float]) -> dict:
    jd_text = job["job_description"]
    jd_embedding = client.embeddings.create(
        input=jd_text, model="text-embedding-3-small"
    ).data[0].embedding
    similarity = cosine_similarity(resume_embedding, jd_embedding)
    job["match_score"] = round(similarity * 100, 1)
    return job

Resume Tailor Agent
Given a job description, customize the resume — reorder skills, mirror JD language, write a targeted summary + cover letter.
Python — agents/tailor.py
TAILOR_SYSTEM_PROMPT = """You are an expert resume coach. Your job is to tailor
a candidate's resume to a specific job description without fabricating experience.

Rules:
1. Mirror keywords from the JD naturally (ATS optimization)
2. Reorder skills to put JD-relevant ones first
3. Emphasize experience bullets most relevant to this role
4. Write a 3-sentence summary targeting this specific role & company
5. Highlight transferable skills if there are gaps
6. NEVER invent experience or inflate titles
7. Flag honestly if the candidate is significantly underqualified

Output: {tailored_resume: ResumeData, match_analysis: str, cover_letter: str}"""

async def tailor_resume_for_job(
    resume: ResumeData,
    job: dict
) -> dict:
    chain = ChatPromptTemplate.from_messages([
        ("system", TAILOR_SYSTEM_PROMPT),
        ("human", """Resume: {resume}
Job Title: {job_title} at {company}
Job Description: {jd}
Requirements: {requirements}""")
    ]) | llm.with_structured_output(TailoredOutput)

    return await chain.ainvoke({
        "resume": resume.model_dump(),
        "job_title": job["job_title"],
        "company": job["employer_name"],
        "jd": job["job_description"],
        "requirements": job["job_required_skills"]
    })

Frontend (Next.js)
4-step wizard UI with real-time streaming, job cards, side-by-side resume diff view.
Step 1
Upload & Parse
Drag-drop PDF upload. Stream parsing progress. Show extracted sections, ATS score gauge (0–100), and a list of gaps with fix suggestions. Allow manual edits inline.
Step 2
Optimize
Side-by-side original vs. improved view with diff highlights. Accept/reject individual bullet rewrites. Re-run to iterate. Show final ATS score improvement delta.
Step 3
Search Jobs
Filter panel (role, location, salary, LOE, remote, tech stack). Card list with company logo, match %, salary range, key requirements. Save to shortlist.
Step 4
Tailor + Export
Click any saved job → stream tailored resume + cover letter. Preview as formatted PDF. One-click download. Track application status.

TypeScript — app/api/optimize/route.ts (streaming)
import { streamText } from 'ai'
import { openai } from '@ai-sdk/openai'

export async function POST(req: Request) {
  const { resume, mode } = await req.json()

  const result = await streamText({
    model: openai('gpt-4o'),
    system: mode === 'optimize'
      ? OPTIMIZE_PROMPT
      : TAILOR_PROMPT,
    messages: [{ role: 'user', content: JSON.stringify(resume) }],
  })

  return result.toDataStreamResponse()
}

SaaS vs. Portfolio
How to position this — open-source portfolio showcase or paid SaaS product.
Portfolio Route
GitHub + Live Demo
Write a strong README with architecture diagram, demo GIF, and tech rationale. Add to portfolio site with a case study. Great for job applications themselves — meta points. Keep it free, add a "Try Demo" with pre-loaded sample resume.
SaaS Route
Freemium Pricing
Free: 1 resume parse + optimize. Pro ($9/mo): unlimited tailoring, cover letters, 50 job searches/day. Add Stripe with Lemon Squeezy for easy billing. Market on LinkedIn, Reddit (r/cscareerquestions), Product Hunt.
Extension
Chrome Extension
Detect job posting on LinkedIn/Indeed → sidebar opens → "Tailor my resume for this job" button. Highest conversion since it's contextual. Build after core product is stable.
